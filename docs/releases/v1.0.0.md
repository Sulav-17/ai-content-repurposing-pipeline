# AI Content Repurposing Pipeline v1.0.0

Release date: 2026-06-22

## Summary

Version 1.0.0 packages the AI Content Repurposing Pipeline as a complete local-first portfolio project. It includes deterministic transcript processing, structured content generation, saved history, local media transcription jobs, Streamlit UI, Docker Compose packaging, CI, evaluation documentation, and release materials.

- Optional local Ollama provider
- PostgreSQL generation history
- Alembic migrations
- Streamlit frontend
- Redis/RQ media jobs
- FFmpeg conversion
- whisper.cpp transcription
- Docker Compose core stack
- Optional Docker media worker profile
- Health, liveness, and readiness endpoints
- Security headers and trusted-host configuration
- Deterministic evaluation harness

## Architecture

The application uses Streamlit over HTTP to FastAPI. FastAPI delegates to service modules for transcript cleaning, analysis, brief generation, platform asset generation, Markdown export, persistence, and media jobs. Providers contain no route logic. PostgreSQL stores saved generation history. Redis/RQ coordinates asynchronous media processing.

## Local-First Approach

The deterministic provider requires no model, paid API, or network access. Ollama is optional and local. Media transcription uses local or containerized FFmpeg and whisper.cpp with a manually supplied model.

## Security Considerations

- No real credentials are committed.
- `.env`, `.env.docker`, generated artifacts, upload data, and model binaries are ignored.
- Model files are supplied manually.
- Containers run as a non-root user.
- Readiness and error responses avoid exposing sensitive configuration.

## Known Limitations

- No authentication or multi-user ownership.
- No public hosting or production deployment is included.
- No social publishing automation.
- No paid AI provider integration.
- No automatic model downloads.
- Generated content requires human review.

## Upgrade Or Installation Notes

This is the first stable release. Create local environment files from the examples and run Alembic migrations before using persistence.
